# Implementation Plan: Floor Plan UX Redesign and App-Wide Design System

**Branch**: `004-app-wide-ux-redesign` | **Date**: 2026-04-20 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/004-app-wide-ux-redesign/spec.md`

## Summary

Fix floor plan editor bugs (chair labels, table labels, uneven chair spacing, drag-vs-pan conflict, missing rotation UI) and implement an app-wide glassmorphism design system with gradient backdrop, full navigation redesign (icons, sections, breadcrumbs), and intuitive UX improvements across all pages (login, dashboard, admin, wedding landing, RSVP, error pages).

## Technical Context

**Language/Version**: TypeScript (strict mode) with Next.js 16 (App Router) + React 19
**Primary Dependencies**: react-konva, konva, react-hook-form, zod, shadcn/ui, Tailwind CSS v4, lucide-react
**Storage**: Supabase PostgreSQL (existing `floor_plans` table — no new migrations)
**Testing**: Vitest + React Testing Library (unit), Playwright (E2E)
**Target Platform**: Web (desktop + mobile Chrome, Safari, Firefox)
**Project Type**: Web application
**Performance Goals**: <200ms perceived interaction, 60fps canvas, glassmorphism without frame drops
**Constraints**: backdrop-filter browser support (96.7%), mobile touch performance, Konva canvas cannot use CSS backdrop-filter
**Scale/Scope**: ~15 pages/views, ~6 floor plan item types, 3 user roles (guest, couple, admin)
